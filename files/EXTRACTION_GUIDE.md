# 일본어 단어 데이터 추출 가이드

## 템플릿 구조

```json
{
  "fileName": "파일명",
  "words": [
    {
      "id": 순번,
      "japanese": "일본어 표기 (한자/히라가나/가타카나)",
      "pronunciation": "발음 (히라가나 또는 가타카나)",
      "meaning": "한국어 의미",
      "sent_jp": "일본어 예문 (원문에서 발췌)",
      "sent_kr": "한국어 예문 번역"
    }
  ]
}
```

## 필드 설명

- **fileName**: 데이터 파일명 (예: "Ado-song", "page_01", "grammar_01")
- **id**: 1부터 시작하는 순차 번호
- **japanese**: 일본어 원형 표기
- **pronunciation**: 정확한 히라가나/가타카나 발음
- **meaning**: 간결하고 명확한 한국어 의미
- **sent_jp**: 해당 단어가 사용된 일본어 예문 (가사/텍스트에서 발췌)
- **sent_kr**: 예문의 자연스러운 한국어 번역

## 작업 절차

1. **텍스트 분석**: 제공된 텍스트에서 학습할 가치가 있는 단어 식별
2. **정보 수집**: 각 단어의 발음과 의미 확인
3. **예문 선택**: 원문에서 해당 단어가 사용된 문장 발췌
4. **번역 작성**: 예문의 자연스러운 한국어 번역 작성
5. **JSON 생성**: 템플릿에 맞춰 데이터 작성

## 주의사항

- 발음은 정확한 히라가나/가타카나로만 표기
- 의미는 문맥에 맞는 가장 적절한 번역 사용
- 예문은 원문의 문장을 그대로 사용 (수정하지 않음)
- 예문 번역은 자연스러운 한국어로 작성
- id는 반드시 1부터 시작하여 순차적으로 부여

## 예시

### 입력 (가사/텍스트)
```
君が瞬きをする音 目を逸らした音さえ
```

## 문법 요소 추출

단어뿐만 아니라 **조사, 접속사, 문법 표현**도 동일한 형식으로 추출합니다.

### 문법 요소 예시

| 문법 | 발음 | 의미 | 예문 |
|------|------|------|------|
| さえ | さえ | 조차, 까지도 | 目を逸らした音さえ |
| だって | だって | ~라도, ~조차 | 寸分の狂いだってない |
| なんて | なんて | ~따위 | 答えなんてさわかりやしないよ |
| ~ようと | ~ようと | ~하든, ~라도 | どんな顔して笑おうと |

### 문법 항목 작성 시 주의사항

- `japanese` 필드에는 문법 형태를 그대로 표기 (예: "さえ", "~だって", "~によって")
- `pronunciation` 필드는 해당 문법의 읽는 법 (대부분 japanese와 동일)
- `meaning` 필드는 문법의 의미/용법을 간결하게 표현
- `sent_jp`는 원문에서 해당 문법이 사용된 문장
- `sent_kr`는 문법의 뉘앙스가 잘 드러나도록 번역
